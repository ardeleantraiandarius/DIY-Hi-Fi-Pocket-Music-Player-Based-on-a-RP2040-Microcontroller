# DIY-Hi-Fi-Pocket-Music-Player-Based-on-a-RP2040-Microcontroller

## Hardware Components
The central processing unit is a Raspberry Pi Pico (RP2040) GroundStudio Marble Pico, chosen because it has an integrated MicroSD slot that helps manage the internal space much better. The audio output relies on a PCM5102 DAC ensuring Hi-Fi sound, capable of decoding up to 384kHz/32-bit audio. Visual feedback is provided by a 0.96-inch monochrome OLED SSD1306 display used for showing device states and track information. Power is handled by a 3.7V 500mAh Li-Po battery charged via a TP4056 USB-C module featuring overcurrent and discharge protection, alongside a simple 3-button user interface.

## Structural Design and Ergonomics
The custom case was consolidated by creating pillars made of hot glue to prevent the plastic from flexing under pressure. Electrical tape was used to cover the edges, protect the internal components, and give it a uniform look. Internal parts were secured using B-7000 adhesive, velcro tape, and hot glue for the final structural fixing. The highly portable device measures 10 cm wide, 5 cm tall, and 2 cm thick, efficiently housing the display, control buttons, 3.5mm audio jack, charging port, and ON/OFF switch. 

## Software Logic and Architecture
The software is written in C++ using the Arduino IDE environment, relying on the ESP8266Audio library for audio processing and Adafruit_GFX for the display interface. To provide a seamless user experience, the program selectively loads only .mp3, .wav, or .flac files during initialization. Data read from the SD card passes through an 8KB RAM buffer to eliminate the risk of audio stuttering or lag caused by SD card read speeds, automatically selecting the appropriate decoder based on the file extension. The GUI indicates the player's status using geometric symbols drawn pixel-by-pixel and features a custom horizontal scrolling algorithm for filenames that exceed the screen width. Track timing is calculated relative to the microcontroller's internal clock, deliberately avoiding the slow process of parsing complex metadata from audio files.
